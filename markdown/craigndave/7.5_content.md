### Decomposition recap
- Decomposition means breaking a complex problem down into smaller, more manageable parts.
- Dealing with many different stages of a problem at once is much more difficult than breaking it down into a number of smaller problems and solving them one at time.

### Step-wise refinement and structure diagrams
- We can use structure diagrams to help decompose a problem.
- We create structure diagrams using an approach known as step-wise refinement.

### Structure diagrams
- Structure diagrams
  - A method of designing a solution to a problem.
  - Produced using a method known as step-wise refinement.
  - Break a problem down into smaller sub-problems using decomposition.
  - Some areas of the program will need to be broken down more than others.
  - Lowest-level nodes should achieve one task – and therefore, be coded as a single module or sub-program.
