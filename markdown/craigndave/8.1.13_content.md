### The importance of local variables
- Subroutines (procedures and functions) have their own local variables.
- Local variables only exist while the subroutine is running.
- Local variables are only accessible from inside the subroutine they are assigned to.
- Benefits of local variables:
  - Help to keep the subroutine self-contained.
  - Subroutines can be used as part of a bigger program without local variables conflicting with other variables of the same name located in other parts of the program.
  - Memory-efficient because the variables only exist (and take up space in memory) while the subroutine is in use.
  - Program will be easier to maintain and debug.

### About global variables
- Variables declared outside of subroutines (procedures and functions) are global variables.
- Global variables exist in memory for the entire time the program is running.
- Global variables are accessible from any subroutine, but this is considered bad practice and should be avoided.
- Benefits of global variables:
  - Memory usage is consistent and known in advance.
  - No additional execution time for reserving and returning memory.
- Drawbacks of global variables:
  - Conflicts with variables that share the same name in other parts of the program makes writing programs in teams and testing more difficult.
  - Memory-inefficient because the variables exist (and take up space in memory) while the program is in use.
- It’s better to pass in copies of global variables to use as parameters when calling subroutines.
