### What are libraries?
- Someone programming in the Windows operating system can call Dynamic Link Libraries (DLL).
- These libraries contain subroutines written to carry out a common task on the Windows operating system – e.g., Save As, where the user needs to save their work as a file.
- All the programmer needs to do is call the appropriate DLL subroutine with the correct parameters, and the Save As dialogue box will appear.

### Benefits and drawbacks of using library routines
- Quick and easy to use and hook into your own code.
- Pre-tested, so you can be relatively sure they are already free from errors.
- Pre-compiled, so they are typically optimised to run quickly.

### Benefits and drawbacks of using library routines
- Adding functionality or making specific tweaks can be difficult – or impossible.
- Quick and easy to use and hook into your own code.
- Sometimes you are black-boxed from the actual implementation.
- Pre-tested, so you can be relatively sure they are already free from errors.
- Pre-compiled, so they are typically optimised to run quickly.
- You have to trust that the developers will continue to maintain the library.
