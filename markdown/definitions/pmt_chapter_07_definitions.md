# Chapter 07 Definitions

*Topic 7: Algorithm design and problem-solving*


## Abnormal Data

Invalid data that should be rejected.


## Abstraction

Removing unnecessary details to focus only on what is important.


## Algorithm

A set of instructions that can be followed to perform a specific task.


## Analysis

The stage where the problem is understood and requirements are identified, using abstraction, decomposition, and identification of inputs, processes, and outputs.


## Boundary Data

Values at the limits (extreme) and just outside the limits.


## Bubble Sort

A sorting algorithm that repeatedly compares and swaps adjacent elements in a series of passes, causing the largest elements to “bubble” to the end with each pass.


## Check Digit

An extra digit calculated from others, used to detect errors.


## Coding

Translating the design into a working program by writing program code and performing iterative testing.


## Counting

Counting how many times a condition is met by increasing a counter each time. Finding Minimum/Maximum: Identifying the smallest or largest value by comparing each new value to the current minimum or maximum.


## Decomposition

Breaking a problem into smaller, more manageable sub-problems.


## Design

Planning the solution before writing code, often using structure diagrams, flowcharts, or pseudocode.


## Double Entry Check

Data is entered twice and compared.


## Dry-Run

Manually going through an algorithm step by step without a computer.


## Extreme Data

Largest and smallest acceptable values.


## Finding Average

Calculating the mean by totalling all values and dividing by the count.


## Flowchart

A diagram showing the step-by-step flow of a process using standard symbols.


## Format Check

Ensures data follows a specific pattern.


## Inputs

Data that goes into a system.


## Length Check

Ensures data has the correct number of characters.


## Linear Search

A searching algorithm that checks items in a list one by one until the desired item is found or the list ends. Education is licensed under CC BY-NC-ND


## Logic Error

An error caused by incorrect logic, such as using the wrong operator or calculation, that will cause an incorrect result despite the program running.


## Normal Data

Typical, valid data within the expected range.


## Outputs

Information produced by the system.


## Presence Check

Ensures data is not left blank.


## Processes

Actions or calculations performed on data.


## Program Code

The actual code written in a specific programming language.


## Program Development Life Cycle

The main stages involved in creating a program: analysis, design, coding, and testing.


## Pseudocode

A text-based, language-independent way of describing a solution in structured steps, using keywords.


## Range Check

Ensures data is within a specified range.


## Runtime Error

An error that occurs while the program is running, such as dividing by zero or using an invalid index.


## Storage

Data saved for later use.


## Structure Diagram

A diagram used to represent decomposition, showing tasks broken down into smaller sub-tasks.


## Syntax Error

An error caused by breaking the rules of the language, such as missing keywords or misspelling commands.


## System Decomposition

Breaking a system into sub-systems to make it easier to design, understand, and test.


## Test Data

Data used to check that a program works correctly.


## Testing

Checking that the program works correctly and meets requirements, using test data.


## Totalling

Adding up a running total of values using a loop.


## Trace Table

A table used during a dry-run to record variable values, outputs, and prompts at each step, helping a user understand or test the flow of a program.


## Type Check

Ensures data is the correct data type.


## Validation

Checking that input data is reasonable and sensible before acceptance.


## Verification

Checking that data matches the original source to prevent data entry errors.


## Visual Check

User compares entered data with the original by eye.
