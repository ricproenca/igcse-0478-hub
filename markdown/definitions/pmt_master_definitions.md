# Master Definitions

*All IGCSE Computer Science definitions organized by chapter*


## Chapter 1: Data Representation

### Analogue Sound

Sound that is continuous and varying in amplitude and frequency, found in the real world.

### ASCII

An early character encoding method, primarily used for English text.

### Binary (Base 2)

A number base using two unique digits (0 and 1), which computers use to represent all data and instructions.

### Bit

The fundamental unit of information, representing either a 0 or a 1.

### Byte

A group of 8 bits.

### Character Set

A defined list of characters that a computer can recognise and use, each mapped to a unique, numerical binary code.

### Colour Depth

The number of bits used to represent the colour of each pixel in an image.

### Data Compression

The process of encoding information using fewer bits than the original representation, to save storage space or reduce transmission time.

### Denary (Base 10)

A number base using ten unique digits (0-9).

### Exbibyte (EiB)

1024 PiB.

### Gibibyte (GiB)

1024 MiB.

### Hexadecimal (Base 16)

A number base using sixteen unique symbols (0-9 and A-F), often used by programmers for its compact representation of binary data.

### Kibibyte (KiB)

1024 bytes.

### Least Significant Bit (MSB)

The bit with the lowest value, which is the rightmost bit, whether it is a 0 or 1, in a binary number.

### Logical Binary Shift

A process that moves bits left (equivalent to multiplying the number by 2 for each place shifted) or right (equivalent to dividing the number by 2 for each place shifted), losing the bits that shift out and adding 0s on the opposite side.

### Lossless Compression

A form of compression where all original information is retained.

### Lossy Compression

A form of compression where some information is lost in the process of reducing the file’s size.

### Mebibyte (MiB)

1024 KiB.

### Metadata (Images)

Data about an image such as: file format, resolution, colour depth, and sometimes details like the device used to capture the image.

### Most Significant Bit (MSB)

The bit with the highest value, which is the leftmost 1 in a binary number.

### Nibble

A group of 4 bits or half a byte

### Overflow Error

An error that occurs when a number or result is too large to be stored in the available number of bits. Two’s Complement: A method of representing signed binary numbers where the MSB is negative. It allows both positive and negative numbers in binary.

### Pebibyte (PiB)

1024 TiB.

### Pixel

Short for "picture element," a single point in an image.

### Resolution

The number of pixels in an image (width x height). Education is licensed under CC BY-NC-ND

### Run Length Encoding (RLE)

A form of data compression that reduces the physical size of a repeating string of characters, by storing the character and the number of times it repeats.

### Sample (Sound)

A measure of the amplitude of an analogue sound wave taken at a specific point in time during the digital conversion process.

### Sample Resolution (Sound)

The number of bits used to store the amplitude of each sample when converting analogue sound to digital.

### Sampling Rate (Sound)

The number of samples taken per second when converting analogue sound to digital, measured in hertz (Hz).

### Tebibyte (TiB)

1024 GiB.

### Unicode

A character encoding standard designed to represent text in all of the world's languages, including non-English alphabets and emojis (more bits per character than ASCII).


## Chapter 2: Data Transmission

### Asymmetric Encryption

An encryption method that uses a public key for encryption and a private key for decryption, allowing secure communication without sharing secret keys.

### Automatic Repeat Query (ARQ)

An error control method where data is resent automatically if an error is detected or no acknowledgment is received within a set time.

### Check Digit

A form of checksum where a single digit, calculated from the data, is added to detect errors in data like ISBNs or barcodes.

### Checksum

An error detection method that uses a calculated value based on the data, which is appended and checked at the destination.

### Ciphertext

The unreadable form of data after encryption, which requires a key to decrypt.

### Data Transmission

The process of sending data from one device to another, often by breaking it into smaller packets.

### Destination address

An IP address that specifies where a packet is going.

### Destination address

An IP address that specifies where a packet came from.

### Echo Check

An error detection method where the receiver sends the received data back to the sender, who compares it to the original.

### Encryption

The process of converting plaintext into unreadable ciphertext to protect data from unauthorised access during storage or transmission.

### Even Parity

A parity system where the number of 1s in the data, including the parity bit, must be even.

### Full-Duplex Transmission

A two-way communication method where data flows in both directions at the same time.

### Half-Duplex Transmission

A two-way communication method where data flows in both directions, but only one direction at a time.

### Odd Parity

A parity system where the number of 1s in the data, including the parity bit, must be odd.

### Packet

A small unit of data used in transmission, containing a header, payload, and trailer.

### Packet Header

Part of a packet that contains control information such as the destination address, originator's address, and packet number.

### Packet number

A sequential identifier assigned to each packet, used to reorder them at the destination.

### Packet Switching

A method of data transmission where data is split into packets, sent separately via the most efficient routes, and reordered at the destination.

### Parallel Transmission

A method of data transmission where multiple bits are sent at the same time using multiple wires or channels.

### Parity Check

An error detection method that uses a parity bit to ensure the number of 1s in a data transmission matches the expected even or odd total.

### Payload

The main content or actual data being sent in a packet.

### Plaintext

The original readable data before encryption.

### Serial Transmission

A method of data transmission where bits are sent one after another over a single wire or channel.

### Simplex Transmission

A one-way communication method where data flows in only one direction.

### Symmetric Encryption

An encryption method where the same private key is used by both sender and receiver to encrypt and decrypt data.

### Trailer

The part of a packet that contains error-checking information to verify data integrity.

### Universal Serial Bus (USB)

A standard interface that connects peripheral devices to a computer, using serial transmission and allowing data and power transfer. Education is licensed under CC BY-NC-ND


## Chapter 3: Hardware

### 2D scanner

An input device that captures flat images or documents as digital files, used to digitize physical documents.

### 3D printer

An output device that builds physical objects layer by layer from digital 3D models using materials like plastic or resin.

### 3D scanner

An input device that captures the shape and appearance of 3D objects to create digital 3D models. Often achieved by taking several pictures from multiple angles.

### Accelerometer

A sensor that measures acceleration, movement, or tilt, capturing motion or orientation data.

### Accumulator (ACC)

A register that stores the results of calculations or operations carried out by the Arithmetic Logic Unit (ALU). Also temporarily holds data being processed.

### Acoustic sensor

A sensor that detects sound or vibrations and measures sound levels.

### Actuator

An output device that produces physical movement, converting digital signals into real-world mechanical actions.

### Arithmetic Logic Unit (ALU)

A component within the CPU responsible for performing arithmetic operations (addition, subtraction, etc.) and logical operations (AND, OR, NOT).

### Barcode scanner

An input device that reads barcodes and converts them into digital data, allowing quick and unique identification of products.

### Cache

A small, fast memory device located on the CPU that stores frequently used data and instructions, providing faster access than main memory.

### Cache Size

The amount of fast, temporary memory (cache) available to the CPU, which stores frequently accessed data for quicker retrieval and affects performance.

### Central Processing Unit (CPU)

The hardware component within a computer that carries out the instructions of a computer program.

### Clock Speed

The rate at which a CPU executes instructions, measured in Hertz (Hz), which affects CPU performance.

### Cloud Storage

A model of data storage where digital data is stored on remote servers, accessed via the internet.

### Control Unit (CU)

A component within the CPU that manages and coordinates the other components of the computer, fetching and decoding instructions.

### Digital camera

An input device that captures photographs or videos as digital files, used to input images into a computer system.

### Digital Light Processing (DLP) projector

An output device that projects images onto a surface using tiny mirrors and light, for large visual displays.

### Embedded System

A computer system with a simple, dedicated function within a larger mechanical or electronic system (e.g., in a washing machine or car).

### Fetch-Decode-Execute Cycle

The fundamental process by which the CPU continually retrieves instructions stored in main memory and decodes and executes them.

### Flow sensor

A sensor that measures the flow rate of liquids or gases.

### Gas sensor

A sensor that detects the presence or concentration of gases in the environment.

### Humidity sensor

A sensor that measures the moisture level in the air.

### Infra-red sensor

A sensor that detects infrared radiation to measure heat, motion, or proximity.

### Inkjet printer

An output device that sprays tiny drops of ink onto paper to produce colour or black-and-white printed images and text.

### Instruction Set

A collection of all the commands (instructions) that a processor can understand and execute, written in machine code.

### IP Address

A unique address used to identify a device on a network. IPv4 uses 32-bit addresses; IPv6 uses 128-bit addresses, allowing for a greater number of unique addresses.

### Keyboard

An input device with keys that allows text, numbers, and commands to be entered into a computer.

### Laser printer

An output device that uses lasers and toner to produce high-speed, high-quality printed output efficiently.

### Level sensor

A sensor that measures the level of liquids or solids in containers.

### Light Emitting Diode (LED) screen

A display that uses LEDs to show images with bright and energy-efficient lighting.

### Light sensor

A sensor that measures the intensity of light.

### Liquid Crystal Display (LCD) projector

An output device that projects images using a light source and liquid crystal panels for visual presentations.

### Liquid Crystal Display (LCD) screen

An output display that uses liquid crystals and a backlight to show images with low power consumption.

### MAC Address

A permanent, unique hardware identifier assigned to a device’s network interface controller at manufacture, used for communication on local networks. Typically written in hexadecimal to reduce their character length.

### Magnetic field sensor

A sensor that detects magnetic fields or changes in magnetism.

### Magnetic Storage

A type of secondary storage that uses magnetic patterns to store data on a rapidly rotating disk (e.g., Hard Disk Drives - HDDs).

### Memory Address Register (MAR)

A register that stores the address to fetch data from or the address where the data is to be stored.

### Memory Data Register (MDR)

A register that stores the data that is being fetched from or written to memory. It acts as a buffer between main memory and the CPU.

### Microphone

An input device that captures analogue sound and converts it into a digital signal for voice or audio input.

### Microprocessor

A type of integrated circuit that contains the CPU (and sometimes other components) on a single chip.

### Moisture sensor

A sensor that detects moisture content in soil or materials.

### Network Interface Card (NIC)

Hardware inside a device that allows it to connect to a network by sending and receiving data signals.

### Number of Processor Cores

The number of independent processing units within a single CPU, allowing for parallel execution of tasks and affecting performance. Education is licensed under CC BY-NC-ND

### Optical mouse

An input device that detects movement and clicks to control a pointer on a screen for navigation and interaction.

### Optical Storage

A type of secondary storage that uses lasers to read and write data on a rotating disc (e.g., CDs, DVDs, Blu-ray discs).

### pH sensor

A sensor that measures the acidity or alkalinity of a solution.

### Pressure sensor

A sensor that detects the force applied by gases or liquids over an area.

### Primary Storage

The computer's workspace for actively running programs, providing fast access to data and instructions currently in use by the CPU.

### Program Counter (PC)

A register that stores the address of the next instruction to be fetched from memory. It increments during each fetch-execute cycle to point to the next instruction.

### Proximity sensor

A sensor that detects the presence or distance of an object without physical contact.

### QR code scanner

An input device that reads QR codes and converts them into readable digital data, such as URLs or payment information.

### RAM (Random Access Memory)

Volatile main memory that can be read from and written to, used for temporary storage of data and programs currently in use by the CPU.

### Register

A small, very fast storage location within the CPU that holds data temporarily during processing.

### ROM (Read-Only Memory)

Non-volatile memory that can only be read from, typically storing essential startup instructions that do not change.

### Router

A device that connects different networks together and directs data between them commonly used to connect LANs to the Internet.

### Secondary Storage

Non-volatile storage mechanisms that are not directly accessible by the CPU, used for long-term, persistent storage of data (e.g., hard drives, SSDs).

### Sensor

A device that detects physical inputs from the environment (e.g. temperature, light, motion and pressure) and converts them into electrical signals.

### Solid State Storage

A type of secondary storage that uses electrical circuits (flash memory, technology, specifically based on NAND or NOR transistors) to persistently store data, with no moving parts.

### Speaker

An output device that converts digital audio signals into analogue sound output like music, voice, or alerts.

### Temperature sensor

A sensor that measures temperature or how hot/cold something is.

### Touch screen (capacitive)

A screen that registers touch using the electrical properties of the finger for more accurate and responsive input.

### Touch screen (infra-red)

A screen that uses a grid of infrared beams to detect touch location and can detect any input object.

### Touch screen (resistive)

A screen that registers touch through pressure on two layers, allowing input by finger or stylus even with gloves.

### Virtual Memory

A memory management technique that allows a computer to run more programs than it has physical RAM (Random Access Memory) by using a portion of the hard drive as if it were RAM.

### Volatile

A type of memory whose contents are lost when the computer loses power.


## Chapter 4: Software

### Application Software

Software that performs specific tasks for the end-user (e.g., word processors or web browsers).

### Assembler

A type of program translator that translates assembly language code into machine code.

### Assembly Language

A low-level programming language that uses mnemonics to represent machine code instructions, which often have a 1:1 correspondence with machine code.

### Bootloader

Part of the firmware that checks that the hardware is working correctly and then loads the operating system into memory.

### Compiler

A type of program translator that translates the entire source code of a program into machine code (an executable file) before the program is run.

### Firmware

Permanent software stored in non-volatile memory (such as ROM) that runs as soon as the computer is powered on. It provides low-level control over the hardware and starts the boot process.

### High-level Language

A programming language that is closer to human language, designed for ease of use, readability, and portability across different computer architectures.

### Integrated Development Environment (IDE)

Software that brings together tools needed to write, test, and debug programs in one place, making development faster and easier.

### Interpreter

A type of program translator that executes source code line by line, without first compiling the entire program into machine code.

### Interrupt

A signal sent to the processor to indicate that an event needs immediate attention.

### Interrupt Service Routine

A block of code that executes in response to a specific interrupt signal. environments (IDEs)

### Low-level Language

A programming language that is closer to machine code and the hardware's instruction set (e.g., machine code, assembly language).

### Machine Code

The lowest-level programming language, specific to a single type of processor, consisting of binary instructions that a computer's CPU can directly execute. Education is licensed under CC BY-NC-ND

### Operating System (OS)

System software that manages the computer's hardware and software resources and provides an interface between the user and device.

### System Software

Software that manages and controls the computer hardware and acts as a platform to run application software.

### Translator

A program that converts source code written in one programming language into another language (often converting programs written in high-level languages into machine code so that they can be executed by a computer’s hardware).

### Utility Programs

System software designed to help maintain, enhance, and troubleshoot a computer system (e.g., disk defragmenter, antivirus).


## Chapter 5: The internet and its uses

### Access Levels

Permissions set within a system to control what data and features different users can access.

### Address Bar

A section of a web browser where users can enter a URL to navigate directly to a website.

### Adware

Software that displays unwanted advertisements and may track user behaviour without consent. It is typically bundled with other programs.

### Anti-malware Software

Programs that detect, quarantine, or remove malicious software by comparing files to known malware databases.

### Authentication

The process of verifying a user’s identity before granting system access, using methods like passwords or biometrics.

### Automated Software Updates

Automatic installation of software patches and fixes to reduce security vulnerabilities.

### Block

A group of transactions recorded together in the blockchain, containing a block hash that links it to the previous block.

### Block Hash

A unique cryptographic code that identifies a block and links it to the previous one in the blockchain.

### Blockchain

A digital ledger consisting of a time-stamped series of records called blocks, which cannot be altered.

### Bookmark

A saved shortcut to a web page stored in a web browser for quick access.

### Brute-force Attack

A method where software tries many combinations of usernames and passwords to guess the correct login credentials.

### Centralised Digital Currency

A digital currency system that is controlled by a central authority such as a bank or government.

### Cookie

A small text file stored on a user’s device by a website to save information such as login details, preferences, and shopping cart contents.

### Data Interception

The unauthorised capturing of data as it is sent over a network, often using special software to listen for unencrypted information such as passwords or credit card numbers.

### Decentralised Digital Currency

A digital currency system that is not controlled by banks or governments.

### Digital Currency

A form of money that exists only in electronic form and has no physical counterpart such as coins or notes. It can be used online to pay for goods or services.

### Digital Signature

An encrypted code used to verify the authenticity of a transaction on the blockchain.

### Distributed Denial of Service (DDoS) attack

An attack that floods a website or online service with excessive traffic from multiple different senders, slowing it down or making it completely unavailable to legitimate users.

### Domain Name

The human-readable address of a website, used instead of an IP address. Web Page / File Name: The specific page or file on the website being requested.

### Domain Name Server (DNS)

A server that translates a website’s domain name into its corresponding IP address.

### Firewall

A system that monitors and controls incoming and outgoing network traffic based on security rules.

### Hacking

Gaining unauthorized access to computer systems or networks by exploiting vulnerabilities or stolen credentials.

### HTTP (Hypertext Transfer Protocol)

A protocol for transferring web pages and other content between web servers and browsers.

### HTTPS (Hypertext Transfer Protocol Secure)

A secure version of HTTP that encrypts communications between a web browser and a website, providing secure web transactions.

### Hypertext Markup Language (HTML)

The language web pages are created in. It gives control over the structure and content of pages, using tags to define elements such as text, images, and links.

### Internet

A global network of interconnected computers and devices. It provides the physical and technical infrastructure that allows data to be sent and received.

### IP Address

A unique numerical address assigned to each device on a network, used to locate and communicate with web servers.

### Malware (Malicious Code)

A general term for harmful software designed to damage, disrupt or gain unauthorised access to computer systems.

### Navigation Tools

Browser buttons such as back, forward, refresh, and stop that help users control webpage loading and browsing history.

### Persistent Cookie

A cookie that remains stored on the device for a set period until it expires or is deleted manually.

### Pharming

An attack that redirects users from a legitimate website to a fake copy in an attempt to steal personal information such as login credentials or bank details.

### Phishing

Fraudulent attempts to obtain private information by pretending to be a trustworthy source, often through email or messages.

### Privacy Settings

Controls that allow users to manage what personal information is shared and who can access it online.

### Protocol

The set of rules that determines how data is transmitted between devices over a network.

### Proxy Server

An intermediary server that hides a user’s IP address and filters web traffic to improve security and privacy.

### Ransomware

Malware that encrypts files or locks users out of their systems, demanding payment for access restoration.

### Secure Socket Layer (SSL)

A security protocol that encrypts data between a web browser and server to prevent interception or tampering.

### Session Cookie

A temporary cookie that is deleted when the web browser is closed. Education is licensed under CC BY-NC-ND

### Social Engineering

The manipulation of people into revealing confidential information by exploiting human psychology, rather than using technical hacking methods.

### Spyware

A type of malware that secretly collects information about a user's activities, such as keystrokes or login details, and sends it to an attacker.

### Tabs

Separate pages opened within a single web browser window that allow multiple web pages to be accessed simultaneously.

### Transaction

A record of the exchange of digital currency between parties, stored in the blockchain ledger.

### Trojan Horse

Malicious software disguised as a legitimate program. Once installed, it can allow hackers to access the system, steal data or install more malware.

### Uniform Resource Locator (URL)

A text-based address assigned to web pages on the internet. It can contain the protocol, domain name and the web page/file name.

### User History

A record maintained by a web browser of all visited URLs along with timestamps for easy revisiting.

### Virus

A type of malware that attaches to legitimate files or programs and spreads when they are opened.

### Web browser

A software application that allows users to access, retrieve, and view content on the World Wide Web.

### World Wide Web (WWW)

A collection of websites and web pages that can be accessed via the internet using web browsers.

### Worm

Malware that spreads independently through networks, by exploiting security flaws, without user action.


## Chapter 6: Automated and emerging technologies

### Ability to Learn and Adapt

Some AI systems improve automatically over time by learning from new data, enabling them to adjust to changing environments without explicit reprogramming.

### Ability to Reason

AI can analyse information, identify patterns, and draw logical conclusions to solve problems and make decisions similar to human reasoning.

### Actuator

A device that takes action based on commands from the microprocessor to physically change something in a real-world system.

### Agricultural Robots

Robots designed to carry out farming tasks such as planting, harvesting, weeding, and monitoring crops to improve productivity and reduce manual labor.

### Artificial Intelligence (AI)

A branch of computer science focused on creating systems that simulate intelligent, often human, behaviour by processing data, reasoning logically, and learning from experience.

### Automated System

A technology-driven mechanism designed to perform tasks with minimal or no human intervention.

### Collection of Data and Rules

### Industry Robots

### Machine Learning

### Medical Robots

### Microprocessor

### Robotics

### Sensor

A device that detects physical inputs from the environment (e.g. temperature, light, motion and pressure) and converts them into electrical signals.

### Transport Robots

Automated machines like driverless cars and delivery drones that operate in transport or logistics to move people or goods without direct human control.

### Abstraction

Removing unnecessary details to focus only on what is important.

### Algorithm

A set of instructions that can be followed to perform a specific task.

### Analysis

The stage where the problem is understood and requirements are identified, using abstraction, decomposition, and identification of inputs, processes, and outputs.

Values at the limits (extreme) and just outside the limits.

### Decomposition

### Double Entry Check

### Dry-Run

Manually going through an algorithm step by step without a computer.

### Extreme Data

Largest and smallest acceptable values.

### Finding Average

Calculating the mean by totalling all values and dividing by the count.

### Flowchart

A diagram showing the step-by-step flow of a process using standard symbols.

### Format Check

Ensures data follows a specific pattern.

### Inputs

Data that goes into a system.

### Length Check

Ensures data has the correct number of characters.

### Linear Search

### Logic Error

### Normal Data

Typical, valid data within the expected range.

### Outputs

Information produced by the system.

### Presence Check

Ensures data is not left blank.

### Processes

Actions or calculations performed on data.

### Program Code

The actual code written in a specific programming language.

### Program Development Life Cycle

The main stages involved in creating a program: analysis, design, coding, and testing.

### Pseudocode

### Range Check

Ensures data is within a specified range.

### Storage

Data saved for later use.

### Structure Diagram

### System Decomposition

### Test Data

### Testing

### Validation

## Chapter 10: Boolean logic

### AND Gate

A logic gate that outputs true only if both its inputs are true.

### Logic Circuit

A set of logic gates that are connected to carry out more complex actions.

### Logic Circuit Diagram

A visual representation using standard symbols for logic gates and their connections to show how a circuit performs a logical function.

### Logic Expression

An expression that represents the logic of a circuit or condition using Boolean operators (AND, OR, NOT, XOR). Education is licensed under CC BY-NC-ND

### Logic Gate

A device that applies logical operations to one or more Boolean inputs to produce a single output.

### NAND Gate

A logic gate that outputs true as long as at least one of its inputs isn’t true.

### NOR Gate

A logic gate that outputs true if neither of its inputs are true, but false otherwise.

### NOT Gate

A logic gate that inverts the input; if the input is true, the output is false, and vice versa.

### OR Gate

A logic gate that outputs true if at least one of its inputs is true.

### Truth Table

A table showing all possible input combinations for a logic gate or circuit and the corresponding output.

### XOR (EOR) Gate

A logic gate that outputs true if exactly one of its inputs is true, but false otherwise.
