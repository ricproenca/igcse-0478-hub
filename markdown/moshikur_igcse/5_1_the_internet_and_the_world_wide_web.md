### **What is the Internet?**

The internet is a large network made up of many smaller networks all over the world. It connects millions of computers and other digital devices, allowing them to communicate and share information.

Think of the internet as a **giant system of roads** that connects cities. Instead of cars, the internet sends data from one place to another. These data include things like emails, websites, videos, and files.

When you use a device like a phone, tablet, or computer to go online, you are using the internet. The internet connects your device to others across the world, such as:

- Websites stored on servers
- Other users
- Online apps and tools

**Key features of the internet:**

- It is **global**, meaning it works in every country.
- It connects many types of devices: computers, mobile phones, TVs, tablets, and more.
- It is made up of cables, satellites, cell towers, routers, and switches.
- It uses **IP addresses** to identify each device that is connected.

---

### **What is the World Wide Web (WWW)?**

The World Wide Web, often called “the web”, is one of the services that runs on the internet. It is a collection of **websites** and **web pages** that you can view using a web browser.

A **web page** is a digital document that contains information such as text, images, videos, or links. A **website** is a group of connected web pages that are stored on a web server.

The web uses something called **HTTP (Hypertext Transfer Protocol)** or **HTTPS** (a secure version of HTTP) to let your browser request and receive web pages.

When you type a web address like `www.moshikur.com`, your browser uses the **internet** to find the server that stores the website and uses **HTTP** to get the web page. The browser then displays it on your screen.

**Example:**

- You type `www.moshikur.com` into your browser.
- Your device sends a request through the internet to find the server that holds the Moshikur website.
- The website sends the requested page back to your browser.
- The browser shows the page so you can read or interact with it.

---

### **What is the Difference Between the Internet and the World Wide Web?**

| Feature | Internet | World Wide Web |
| --- | --- | --- |
| Type | A network of networks | A service that runs on the internet |
| Function | Allows computers to connect and send data | Lets users access and view websites |
| Includes | Devices, cables, routers, and IP addresses | Web pages, websites, hyperlinks |
| Example | Sending an email, downloading a file | Reading an article on a website |

- The internet is the **highway system**.
- The World Wide Web is like the **cars and trucks** (web pages and websites) that travel on the highways.

# **5.1.2 The Role of Web Browsers and URLs**

[![Diagram illustrating the components of a URL, including 'protocol', 'domain name', and 'web page/file name'.](images/topic5_internet_and_uses/5_1_the_internet_and_the_world_wide_web/img_002.png)](https://i0.wp.com/moshikur.com/wp-content/uploads/2025/09/image-102.png?ssl=1)

### **What is a Web Browser?**

A web browser is a type of software that lets users access and view web pages on the internet. It translates code (like HTML and CSS) from websites into text, images, videos, and links that people can understand and interact with.

Common web browsers include:

- Google Chrome
- Mozilla Firefox
- Microsoft Edge
- Safari

These examples are for your understanding only. You should not write any brand names in the exam.

Web browsers are installed on computers, tablets, and phones. They are essential tools for exploring websites.

---

### **Main Functions of a Web Browser**

## **Displaying Websites**

The browser fetches the code of a web page and shows it in a user-friendly way. It interprets languages like HTML and CSS so the page appears correctly.

## **Storing Cookies**

Cookies are small text files that websites use to remember users. Browsers store these cookies to save login details, preferences, and shopping cart items.

## **Bookmarking Web Pages**

Users can save websites they like by bookmarking them. This allows quick access later without typing the full web address again.

## **Auto-Fill Information**

Browsers can remember and fill in personal details such as names, email addresses, and payment information on forms.

## **Managing History**

Browsers keep a record of all the websites that were visited. This helps users return to previous sites easily.

## **Downloading Files**

A browser can download files such as images, PDFs, and software from the internet onto a device.

---

### **What is a URL?**

URL stands for **Uniform Resource Locator**. It is the address of a web page. Every page on the internet has a unique URL. When you type a URL into the address bar of a browser, it finds and displays that web page.

Example:  
`https://www.moshikur.com/index.html`

---

### **Parts of a URL**

A URL usually has several parts:

1. **Protocol**  
   This tells the browser how to communicate with the server.  
   Example: `http` or `https`
2. **Domain Name**  
   This is the name of the website.  
   Example: `moshikur.com`
3. **Path**  
   This shows the exact page or file on the website.  
   Example: `/index.html`

So the full URL explains what to access, where to find it, and how to get it.

## **What is DNS?**

DNS stands for **Domain Name System**.

DNS is a system that **translates a website’s URL (domain name)** into an **IP address** so that the correct **web server** can be found.

---

## **Why DNS is Needed**

- Computers **do not understand domain names**
- Computers communicate using **IP addresses**
- DNS acts like an **internet phone book**

# 5.1.3 How Web Pages Are Located, Retrieved, and Displayed

When you type a web address (URL) into your web browser, a lot of things happen in the background to display the correct web page on your screen. This process involves several components working together: the web browser, DNS, IP addresses, and the web server.

Let’s break it down step-by-step.

---

### Step 1: You Enter a URL

You start by typing a web address (URL) like:

```
https://www.moshikur.com/notes.html
```

This tells your browser which page you want to see.

---

### Step 2: The Browser Uses DNS to Find the IP Address

Each website on the internet is stored on a computer called a **web server**. To reach that server, your browser needs to know its **IP address** (a unique number like `192.168.1.5`).

But humans use domain names like `www.moshikur.com`, not numbers.

To solve this, your browser contacts a **Domain Name Server (DNS)**. The DNS translates the domain name into the correct IP address.

For example:

- `www.moshikur.com` → `165.242.75.10`

---

### Step 3: The Browser Sends a Request to the Web Server

Once the IP address is known, your browser sends a **request** to the web server at that IP address. This request asks for the specific file or page (like `notes.html`).

The request is sent using a protocol, usually **HTTP** or **HTTPS**.

---

### Step 4: The Web Server Responds

The web server receives the request. If the requested page exists and the user has permission to view it, the server sends back the web page’s content.

This content is usually written in **HTML** (Hypertext Markup Language).

---

### Step 5: The Browser Displays the Page

Your web browser receives the HTML file and **renders** it — this means it turns the HTML code into something visual.

It displays:

- Text
- Images
- Videos
- Links
- Buttons
- Forms

This is what you see on your screen.

---

### Example Flow

Let’s go through a full example:

1. You type: `https://www.moshikur.com/notes.html`
2. Browser checks with DNS → Gets IP address
3. Browser sends HTTP request to server
4. Server sends back HTML code for the page
5. Browser renders and displays the page

---

| Term | Description |
| --- | --- |
| URL | The address of a web page |
| DNS | Translates domain names into IP addresses |
| IP Address | A unique number that identifies a device on a network |
| HTTP/HTTPS | Protocol used to request and transfer web pages |
| Web Server | A computer that stores and sends web pages |
| HTML | The language used to structure web content |
| Browser | Software that requests, receives, and displays web pages |

## 5.1.4 Cookies and Their Use

[![An infographic about cookies and their use, featuring a green cookie graphic, a shopping cart, and a user profile icon. The text lists three points: 'Session cookies', 'Persistent cookies', and 'Tracking preferences'. The website URL 'www.moshikur.com' is included at the bottom.](images/topic5_internet_and_uses/5_1_the_internet_and_the_world_wide_web/img_003.png)](https://i0.wp.com/moshikur.com/wp-content/uploads/2025/09/image-103.png?ssl=1)

Cookies are small text files that are stored on your computer or device by a website. They are used to remember information about you when you visit a website.

Cookies are not programs or viruses. They do not run any code, and they do not harm your device. Instead, they store simple data, like your login status or what items you added to a shopping cart.

---

### Why Websites Use Cookies

Websites use cookies to improve the user experience. When you visit a website again, cookies help the website remember who you are and what you did last time.

Common uses of cookies:

- Remember your username and keep you logged in
- Store items in your shopping cart
- Save your language preference
- Track your activity for personalised ads or content

---

### Types of Cookies

There are two main types of cookies you need to know for your IGCSE/O Level syllabus:

## 1. Session Cookies

- These are temporary cookies.
- They are created when you open a website and deleted when you close the browser.
- They store short-term data, such as:
  - Items added to a cart
  - Navigation history during a session
- Example: On an online shopping site, a session cookie remembers what you selected until you check out or close the page.

## 2. Persistent Cookies

- These cookies stay on your device even after you close the browser.
- They are stored for a set time or until the user deletes them.
- They store long-term data, such as:
  - Login details
  - Preferences (like language or theme)
  - Analytics for website visits
- Example: When a website remembers your login so you don’t have to type it every time, that’s done using a persistent cookie.

---

### Where Are Cookies Stored?

Cookies are saved by the web browser on the local device. Each browser stores cookies in a separate location, but all of them allow websites to access and update these cookies when needed.

---

### Important Points

- Cookies are linked to a specific website.
- Only the website that created the cookie can read it.
- Websites must ask permission before using certain types of cookies (especially for tracking).

# 5.1.5 Web Page Security

[![Infographic explaining the components of a URL, including protocol, domain name, and file path, with emphasis on web page security.](images/topic5_internet_and_uses/5_1_the_internet_and_the_world_wide_web/img_004.png)](https://i0.wp.com/moshikur.com/wp-content/uploads/2025/09/image-105.png?ssl=1)

---

When you use the internet to send personal information — like passwords, credit card numbers, or private messages — that data must be **protected** so that no one else can read or steal it. This is where **SSL** comes in.

---

### What is SSL?

**SSL** stands for **Secure Sockets Layer**. It is a **security protocol** used to create a secure and encrypted connection between your browser and the web server.

Without SSL, any data sent over the internet can be intercepted and read by hackers. With SSL, the data is encrypted so that it becomes unreadable to anyone except the intended server.

---

### Why Is SSL Important?

SSL protects users when they:

- Log in to accounts
- Make online purchases
- Fill out forms with personal details

It keeps data **private**, **safe**, and **unreadable** to third parties during transmission.

---

### How SSL Works (in simple steps)

1. **Browser requests a secure connection**  
   When you visit a website using **HTTPS**, your browser asks the web server to create a secure connection.
2. **The server sends its SSL certificate**  
   This certificate contains the server’s identity and a **public key** used for encryption. It is like a digital ID card for the server.
3. **Browser checks the certificate**  
   The browser checks whether the SSL certificate is **valid**, **trusted**, and **not expired**.
4. **Secure connection is created**  
   If everything is correct, a secure, encrypted connection is established. Any data you send (like passwords) is protected.

---

### What is an SSL Certificate?

An **SSL certificate** is a digital file used to verify the identity of a website. It is issued by a **Certificate Authority (CA)** — a trusted company that confirms that the website is real and secure.

The certificate includes:

- The domain name
- The organisation name
- The certificate’s expiry date
- The public key used to encrypt data

---

### HTTPS and SSL

Websites that use SSL show **HTTPS** in their address instead of HTTP. The **S** stands for **Secure**.

Example:

- Insecure: `http://www.example.com`
- Secure: `https://www.example.com`

Most browsers also show a **padlock icon** in the address bar to let users know the website is protected with SSL.

---

### Summary of Key Points

| Feature | Description |
| --- | --- |
| SSL | Protocol that secures data sent between browser and web server |
| Encryption | Converts data into unreadable format to protect it |
| SSL Certificate | Proves the identity of the website and enables encryption |
| HTTPS | Secure version of HTTP using SSL |
| Padlock Icon | Appears in browser to show SSL is active |

# 5.1.6 Web Page Addresses

[![Illustration explaining the structure of a URL, including the protocol, domain name, and web page/file name.](images/topic5_internet_and_uses/5_1_the_internet_and_the_world_wide_web/img_005.png)](https://i0.wp.com/moshikur.com/wp-content/uploads/2025/09/image-104.png?ssl=1)

## Structure of a URL

Every web page on the internet has a unique **URL**, which stands for **Uniform Resource Locator**. A URL is the address you type into your browser to visit a specific page on a website.

Think of a URL like a postal address. Just as your home has a street name, house number, and city, a URL has parts that tell the browser where to go and what to request.

---

### Example of a URL

```
https://www.moshikur.com/resources/lesson1.html
```

This URL points to a specific file called `lesson1.html` in the `resources` folder of the website `moshikur.com`.

---

### Structure of a URL

A URL is made up of several parts. Each part has a specific purpose.

| Part | Description | Example |
| --- | --- | --- |
| Protocol | Tells the browser how to communicate with the server | https:// |
| Domain Name | The name of the website or server | www.moshikur.com |
| Folder Path | Shows the location of the file within folders on the server | /resources/ |
| File Name | The name of the specific page or file being requested | lesson1.html |

---

### Description of Each Part

#### 1. **Protocol**

This is usually `http://` or `https://`.

- **HTTP**: Hypertext Transfer Protocol (not secure)
- **HTTPS**: Secure version of HTTP, uses SSL encryption

Web browsers use the protocol to know **how** to request and transfer data from the web server.

#### 2. **Domain Name**

This is the name of the website, like `www.moshikur.com`.  
The browser uses this name to contact a **DNS (Domain Name Server)** to get the IP address of the website’s server.

#### 3. **Folder Path**

Just like folders on your computer, websites store files in folders.  
In the example, `resources` is a folder that holds the file.

#### 4. **File Name**

The name of the file being requested. This is usually an HTML file, like `lesson1.html`.

---

### Another Example

```
http://example.com/blog/post1.html
```

| Part | Value |
| --- | --- |
| Protocol | http |
| Domain Name | example.com |
| Folder Path | /blog/ |
| File Name | post1.html |

---

### Important Notes

- All URLs must be **unique** so that the browser can locate the exact web page.
- If the file name is not included, the browser usually loads a default file like `index.html`.
- URLs should not have spaces or special characters.
