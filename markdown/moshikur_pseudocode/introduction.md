# **1.1 What Is Pseudocode?**

Pseudocode is a **way of writing algorithms using plain English and simple programming-style statements**.

It is **not a real programming language**.  
Instead, it helps us **plan and explain solutions** to problems before writing actual code.

Pseudocode focuses on:

### **Logic**

### **Steps**

##### **Order of actions**

and **not** on the rules of a specific programming language like Python or Java.

---

# **1.2 Why Do We Use Pseudocode?**

Pseudocode is used because it is:

- Easy to **read**
- Easy to **understand**
- Easy to **convert into any programming language**

In Cambridge IGCSE and O Level Computer Science:

- You are **not tested on a programming language**
- You are tested on **logical thinking and problem solving**

That is why pseudocode is used in exams.

---

# **1.3**

# **Pseudocode in the Cambridge Exam**

Cambridge provides an **official pseudocode guide** that must be followed in exams.

This means:

- Keywords must be written in **capital letters**
- Indentation must be correct
- Only **allowed commands and functions** can be used

If you use:

- Real programming code
- Commands not in the guide

you **may lose marks**, even if your logic is correct.

---

## **1.4 Pseudocode vs Programming Language**

| Pseudocode | Programming Language |
| --- | --- |
| Not executed by a computer | Executed by a computer |
| Uses simple English | Uses strict syntax |
| Used in exams | Used to build software |
| Focuses on logic | Focuses on syntax + logic |

## Example:

Pseudocode:

```
OUTPUT "Hello"
```

Programming language (Python):

```
print("Hello")
```

In exams, **only pseudocode is accepted**.

---

## **1.5 What Can Be Written Using Pseudocode?**

Using pseudocode, we can write algorithms to:

- Input data
- Output results
- Make decisions
- Repeat steps using loops
- Store data in arrays
- Use procedures and functions
- Read from and write to files

All of these are **part of the IGCSE/O Level syllabus**.

---

## **1.6 What Pseudocode Is NOT**

Pseudocode is **not**:

- Python
- Java
- C++
- A flowchart
- Real executable code

You should **never mix programming syntax** with pseudocode in exams.

❌ Incorrect:

```
print("Enter number")
```

✅ Correct:

```
OUTPUT "Enter number"
```

# **1.7 Why Learning Pseudocode Is Important**

Learning pseudocode helps you to:

- Think logically
- Break problems into steps
- Score well in **Paper 2**
- Easily learn programming later

If you understand pseudocode well, **programming becomes much easier**.

- Pseudocode is a **planning tool**
- It uses **simple English**
- It follows **Cambridge rules**
- It focuses on **logic, not syntax**

# Pseudocode Rules and Conventions

## **1 Why Rules Matter in Pseudocode**

In Cambridge exams, pseudocode is **not free writing**.

Cambridge provides a **fixed set of rules** for how pseudocode must be written.  
If these rules are not followed, **marks can be lost**, even if the logic is correct.

So, learning the rules is just as important as learning the logic.

---

## **2 Keywords in Pseudocode**

Keywords are **special words** that tell the computer what to do.

In Cambridge pseudocode:

- **All keywords must be written in CAPITAL letters**

Examples of keywords:

INPUT

OUTPUT

IF

THEN

ELSE

ENDIF

FOR

WHILE

✅ Correct:

```
IF Number > 10 THEN
```

❌ Incorrect:

```
if Number > 10 then
```

---

## **3 Indentation**

Indentation means **spacing at the start of a line**.

Indentation shows which statements belong together.

Cambridge expects:

- Statements inside IF, loops, procedures, etc. to be **indented**
- Clear structure so the logic is easy to follow

Example:

```
IF Number > 0 THEN

OUTPUT Number

ENDIF
```

Poor indentation can make your answer **hard to read** and may lose marks.

---

## **4 Identifiers (Variable Names)**

Identifiers are the **names given to variables, arrays, procedures, and functions**.

Rules for identifiers:

- Use **letters and numbers only**
- Start with a **capital letter**
- Do not start with a number
- Do not use spaces
- Do not use keywords as names

✅ Correct:

```
TotalMarks

StudentAge

Count
```

❌ Incorrect:

```
total marks

1Score

IF
```

Choose names that **describe what the variable stores**.

## **5 Comments**

Comments are used to **explain what the code is doing**.

In Cambridge pseudocode:

- Comments start with `//`
- Comments are **ignored by the computer**

Example:

```
// This program counts positive numbers
```

Comments are helpful but:

- Do not explain obvious code
- Do not write essays in comments

---

## **6 Line Structure**

Each instruction should normally be written on **one line**.

Avoid:

- Very long lines
- Multiple actions on one line

Clear, short statements are easier for examiners to understand.

---

## **7 What You Must NOT Do**

In exams, you must **not**:

- Use real programming code (Python, Java, etc.)
- Invent your own commands
- Change keyword spelling
- Use unsupported functions

❌ Incorrect:

```
print("Hello")
```

✅ Correct:

```
OUTPUT "Hello"
```

---

## **8 Common Exam Mistakes**

Students often lose marks by:

- Writing keywords in lowercase
- Forgetting indentation
- Mixing programming languages with pseudocode
- Using commands not in the pseudocode guide

Avoid these mistakes to **protect easy marks**.

---

## **9 Key Points to Remember**

- Keywords are in **CAPITAL letters**
- Indentation shows structure
- Variable names must follow rules
- Use `//` for comments
- Follow the Cambridge pseudocode guide exactly
