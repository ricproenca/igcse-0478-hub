## Student-Facing Website Features

### Topic Pages
One unified revision page per syllabus topic, combining content from all available sources: textbook notes, Craig 'n' Dave slide summaries, Save My Exams notes, and PMT notes. Gives students a single place to study each topic without switching between resources.
**Source**: all `markdown/` folders

### Definition Glossary
A-Z searchable glossary of all key terms drawn from the 10 chapter definition files. Essential for quick look-ups and last-minute revision.
**Source**: `markdown/definitions/`

### Pseudocode Reference Card
Quick-reference cheat sheet covering all pseudocode syntax, operators, and constructs required by the syllabus. Ideal for programming questions on Paper 2.
**Source**: `markdown/moshikur_pseudocode/`

### Flashcard Sets
One flashcard set per topic, generated from the Save My Exams flashcard content. Exportable to Anki or Quizlet for spaced repetition practice.
**Source**: `markdown/savemyexams_flashcards/`

### Examiner Tips Compilation
All "Examiner Tips and Tricks" callouts extracted from Save My Exams notes, grouped by topic. Highlights the most common mistakes and what markers look for.
**Source**: `markdown/savemyexams/`

### Syllabus Progress Tracker
Interactive checklist of all syllabus topics so students can mark what they've studied, what needs review, and what they feel confident on.

## New Ideas (from content analysis)

### Step-by-Step Worked Examples Drill
The PMT notes contain detailed conversion walkthroughs — binary↔denary↔hex — shown step-by-step using place-value tables. These could become interactive "reveal the next step" exercises: the student attempts each step, then checks against the worked solution. Covers number systems, two's complement, binary shifts, and image/sound file size calculations.
**Source**: `markdown/notes/`

### Interactive Place-Value Table Tool
Chapter 1 content (textbook + PMT notes) is rich in place-value tables (powers of 2, 16, etc.). A fill-in-the-blank table tool where students enter a number and progressively complete the breakdown would make number conversion practice much more hands-on than reading static notes.
**Source**: `markdown/notes/`, `markdown/textbook/`

### Topic Learning Path Map
The moshikur IGCSE content is structured hierarchically (1.1.1 → 1.1.2 → 1.1.3 within each topic) and maps sub-topic dependencies clearly. This structure could become a visual learning path diagram — showing students the recommended order to study sub-topics and which concepts build on each other before moving forward.
**Source**: `markdown/moshikur_igcse/`

### IGCSE vs A-Level Pseudocode Comparison
The moshikur pseudocode content explicitly labels each construct as "IGCSE/O Level" or "A Level" — covering operators, library routines, functions, arrays, file handling, and more. This could become a side-by-side comparison guide for students planning to continue to AS/A Level, showing exactly what changes at the next stage.
**Source**: `markdown/moshikur_pseudocode/`

### Real-World Context Cards
Craig 'n' Dave presentation content consistently explains the practical *why* behind each concept — e.g., "two states = easier to manufacture, cheaper, more reliable." These "why it matters" explanations are often missing from dry revision notes. A "real-world context" card per topic, pulled from the Craig 'n' Dave summaries, would help students understand rather than just memorise.
**Source**: `markdown/craigndave/`

### Flashcard Q&A Extractor
The Save My Exams flashcard PDFs contain numbered front/back Q&A pairs embedded in the text (e.g., "Why does data have to be converted to binary? / Because computers are built using switches..."). These pairs exist across 30 topics and ~400+ cards total. Properly parsing and rendering them as web flashcards (with flip animation) would be more effective than the current image-heavy PDF format.
**Source**: `markdown/savemyexams_flashcards/`

### Multi-Source Coverage Guide
Each topic is covered by up to five different sources at different depths and styles. A meta-guide per topic — showing "textbook (detailed theory), Craig 'n' Dave (visual summary), Save My Exams (exam-focused), PMT (advanced notes), moshikur (worked examples)" — would help students choose the right resource for their current need (first learning vs. quick revision vs. exam practice).
**Source**: all `markdown/` folders
